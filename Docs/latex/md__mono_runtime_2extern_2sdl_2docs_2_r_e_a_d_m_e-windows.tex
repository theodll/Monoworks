SDL can be built with Min\+GW-\/w64 and CMake. Minimum tested Min\+GW-\/w64 version is 8.\+0.\+3.

On a Windows host, you first need to install and set up the MSYS2 environment, which provides the Min\+GW-\/w64 toolchain. Install MSYS2, typically to {\ttfamily C:\+\textbackslash{}msys64}, and follow the instructions on the MSYS2 wiki to use the Min\+GW-\/w64 shell to update all components in the MSYS2 environment. This generally amounts to running {\ttfamily pacman -\/Syuu} from the mingw64 shell, but refer to MSYS2\textquotesingle{}s documentation for more details. Once the MSYS2 environment has been updated, install the x86\+\_\+64 Min\+GW toolchain from the mingw64 shell with the command {\ttfamily pacman -\/S mingw-\/w64-\/x86\+\_\+64-\/toolchain}. (You can additionally install {\ttfamily mingw-\/w64-\/i686-\/toolchain} if you intend to build 32-\/bit binaries as well. The remainder of this section assumes you only want to build 64-\/bit binaries.)

To build and install SDL, you can use Power\+Shell or any CMake-\/compatible IDE. First, install CMake, Ninja, and Git. These tools can be installed using any number of tools, such as the MSYS2\textquotesingle{}s {\ttfamily pacman}, {\ttfamily winget}, {\ttfamily Chocolatey}, or by manually downloading and running the installers. Clone SDL to an appropriate location with {\ttfamily git} and run the following commands from the root of the cloned repository:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir\ build}
\DoxyCodeLine{cmake\ -\/S\ .\ -\/B\ build\ -\/G\ Ninja\ -\/DCMAKE\_TOOLCHAIN\_FILE=build-\/scripts/cmake-\/toolchain-\/mingw64-\/x86\_64.cmake}
\DoxyCodeLine{cmake\ -\/-\/build\ build\ -\/-\/parallel}
\DoxyCodeLine{cmake\ -\/-\/install\ build\ -\/-\/prefix\ C:/Libraries}

\end{DoxyCode}


This installs SDL to {\ttfamily C:\+\textbackslash{}Libraries}. You can specify another directory of your choice as desired. Ensure that your {\ttfamily CMAKE\+\_\+\+PREFIX\+\_\+\+PATH} includes {\ttfamily C:\+\textbackslash{}Libraries} when you want to build against this copy of SDL. The simplest way to do this is to pass it to CMake as an option at configuration time:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ ..\ -\/G\ Ninja\ -\/DCMAKE\_PREFIX\_PATH=C:/Libraries}

\end{DoxyCode}


You will also need to configure CMake to use the Min\+GW-\/w64 toolchain to build your own project. Here is a minimal toolchain file that you could use for this purpose:\+

\DoxyCodeLine{set(CMAKE\_SYSTEM\_NAME\ Windows)}
\DoxyCodeLine{set(CMAKE\_SYSTEM\_PROCESSOR\ x86\_64)}
\DoxyCodeLine{}
\DoxyCodeLine{find\_program(CMAKE\_C\_COMPILER\ NAMES\ x86\_64-\/w64-\/mingw32-\/gcc\ REQUIRED)}
\DoxyCodeLine{find\_program(CMAKE\_CXX\_COMPILER\ NAMES\ x86\_64-\/w64-\/mingw32-\/g++\ REQUIRED)}
\DoxyCodeLine{find\_program(CMAKE\_RC\_COMPILER\ NAMES\ x86\_64-\/w64-\/mingw32-\/windres\ windres\ REQUIRED)}

\end{DoxyCode}


Save this in your project and refer to it at configuration time with the option {\ttfamily -\/DCMAKE\+\_\+\+TOOLCHAIN\+\_\+\+FILE}.

On Windows, you also need to copy {\ttfamily SDL3.\+dll} to an appropriate directory so that the game can find it at runtime. For guidance, see \href{README-cmake.md\#how-do-i-copy-a-sdl3-dynamic-library-to-another-location}{\texttt{README-\/cmake.\+md}}.

\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-windows_autotoc_md1483}{}\doxysection{\texorpdfstring{Open\+GL ES 2.\+x support}{Open\+GL ES 2.\+x support}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-windows_autotoc_md1483}
SDL has support for Open\+GL ES 2.\+x under Windows via two alternative implementations.

The most straightforward method consists in running your app in a system with a graphic card paired with a relatively recent (as of November of 2013) driver which supports the WGL\+\_\+\+EXT\+\_\+create\+\_\+context\+\_\+es2\+\_\+profile extension. Vendors known to ship said extension on Windows currently include n\+Vidia and Intel.

The other method involves using the \href{https://code.google.com/p/angleproject/}{\texttt{ANGLE library}}. If an Open\+GL ES 2.\+x context is requested and no WGL\+\_\+\+EXT\+\_\+create\+\_\+context\+\_\+es2\+\_\+profile extension is found, SDL will try to load the lib\+EGL.\+dll library provided by ANGLE.

To obtain the ANGLE binaries, you can either compile from source from \href{https://chromium.googlesource.com/angle/angle}{\texttt{https:\+/\+/\+chromium.\+googlesource.\+com/\+angle/\+angle}} or copy the relevant binaries from a recent Chrome/\+\+Chromium install for Windows. The files you need are:\+


\begin{DoxyItemize}
\item lib\+EGL.\+dll
\item lib\+GLESv2.\+dll
\item d3dcompiler\+\_\+46.\+dll (supports Windows Vista or later, better shader compiler) {\itshape or} d3dcompiler\+\_\+43.\+dll (supports Windows XP or later)
\end{DoxyItemize}

If you compile ANGLE from source, you can configure it so it does not need the d3dcompiler\+\_\+\texorpdfstring{$\ast$}{*} DLL at all (for details on this, see their documentation). However, by default SDL will try to preload the d3dcompiler\+\_\+46.\+dll to comply with ANGLE\textquotesingle{}s requirements. If you wish SDL to preload d3dcompiler\+\_\+43.\+dll (to support Windows XP) or to skip this step at all, you can use the SDL\+\_\+\+HINT\+\_\+\+VIDEO\+\_\+\+WIN\+\_\+\+D3\+DCOMPILER hint (see \doxylink{_s_d_l__hints_8h_source}{SDL\+\_\+hints.\+h} for more details).

Known Bugs:\+


\begin{DoxyItemize}
\item SDL\+\_\+\+GL\+\_\+\+Set\+Swap\+Interval is currently a no op when using ANGLE. It appears that there\textquotesingle{}s a bug in the library which prevents the window contents from refreshing if this is set to anything other than the default value.
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-windows_autotoc_md1484}{}\doxysection{\texorpdfstring{Vulkan Surface Support}{Vulkan Surface Support}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-windows_autotoc_md1484}
Support for creating Vulkan surfaces is configured on by default. To disable it change the value of {\ttfamily SDL\+\_\+\+VIDEO\+\_\+\+VULKAN} to 0 in {\ttfamily SDL\+\_\+config\+\_\+windows.\+h}. You must install the \href{https://www.lunarg.com/vulkan-sdk/}{\texttt{Vulkan SDK}} in order to use Vulkan graphics in your application.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-windows_autotoc_md1485}{}\doxysection{\texorpdfstring{Transparent Window Support}{Transparent Window Support}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-windows_autotoc_md1485}
SDL uses the Desktop Window Manager (DWM) to create transparent windows. DWM is always enabled from Windows 8 and above. Windows 7 only enables DWM with Aero Glass theme.

However, it cannot be guaranteed to work on all hardware configurations (an example is hybrid GPU systems, such as NVIDIA Optimus laptops).
