HIDAPI is a multi-\/platform library which allows an application to interface with USB and Bluetooth HID-\/\+Class devices on Windows, Linux, Free\+BSD, and mac\+OS. HIDAPI can be either built as a shared library ({\ttfamily .so}, {\ttfamily .dll} or {\ttfamily .dylib}) or can be embedded directly into a target application by adding a {\itshape single source} file (per platform) and a single header.~\newline
 See \href{BUILD.md\#embedding-hidapi-directly-into-your-source-tree}{\texttt{remarks}} on embedding {\itshape directly} into your build system.

HIDAPI library was originally developed by Alan Ott (\href{https://github.com/signal11}{\texttt{signal11}}).

It was moved to \href{https://github.com/libusb/hidapi}{\texttt{libusb/\+hidapi}} on June 4th, 2019, in order to merge important bugfixes and continue development of the library.\hypertarget{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1660}{}\doxysubsection{\texorpdfstring{Table of Contents}{Table of Contents}}\label{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1660}

\begin{DoxyItemize}
\item Windows (using {\ttfamily hid.\+dll})
\item Linux/\+hidraw (using the Kernel\textquotesingle{}s hidraw driver)
\item libusb (using libusb-\/1.\+0 -\/ Linux/\+\+BSD/\+other UNIX-\/like systems)
\item mac\+OS (using IOHid\+Manager)
\end{DoxyItemize}

On Linux, either the hidraw or the libusb back-\/end can be used. There are tradeoffs, and the functionality supported is slightly different. Both are built by default. It is up to the application linking to hidapi to choose the backend at link time by linking to either {\ttfamily libhidapi-\/libusb} or {\ttfamily libhidapi-\/hidraw}.

Note that you will need to install an udev rule file with your application for unprivileged users to be able to access HID devices with hidapi. Refer to the \href{udev/69-hid.rules}{\texttt{69-\/hid.\+rules}} file in the {\ttfamily udev} directory for an example.\hypertarget{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1663}{}\doxysubsubsubsection{\texorpdfstring{{\bfseries{Linux/\+hidraw}} ({\ttfamily linux/\+hid.\+c}):\+}{{\bfseries{Linux/\+hidraw}} ({\ttfamily linux/\+hid.\+c}):\+}}\label{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1663}
This back-\/end uses the hidraw interface in the Linux kernel, and supports both USB and Bluetooth HID devices. It requires kernel version at least 2.\+6.\+39 to build. In addition, it will only communicate with devices which have hidraw nodes associated with them. Keyboards, mice, and some other devices which are blacklisted from having hidraw nodes will not work. Fortunately, for nearly all the uses of hidraw, this is not a problem.\hypertarget{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1664}{}\doxysubsubsubsection{\texorpdfstring{{\bfseries{Linux/\+\+Free\+BSD/\+libusb}} ({\ttfamily libusb/\+hid.\+c}):\+}{{\bfseries{Linux/\+\+Free\+BSD/\+libusb}} ({\ttfamily libusb/\+hid.\+c}):\+}}\label{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1664}
This back-\/end uses libusb-\/1.\+0 to communicate directly to a USB device. This back-\/end will of course not work with Bluetooth devices.\hypertarget{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1665}{}\doxysubsubsection{\texorpdfstring{Test GUI}{Test GUI}}\label{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1665}
HIDAPI also comes with a Test GUI. The Test GUI is cross-\/platform and uses Fox Toolkit \href{http://www.fox-toolkit.org}{\texttt{http:\+/\+/\+www.\+fox-\/toolkit.\+org}}. It will build on every platform which HIDAPI supports. Since it relies on a 3rd party library, building it is optional but it is useful when debugging hardware.

NOTE:\+ Test GUI based on Fox Toolkit is not actively developed nor supported by HIDAPI team. It is kept as a historical artifact. It may even work sometime or on some platforms, but it is not going to get any new features or bugfixes.

Instructions for installing Fox-\/\+Toolkit on each platform is not provided. Make sure to use Fox-\/\+Toolkit v1.\+6 if you choose to use it.\hypertarget{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1666}{}\doxysubsubsection{\texorpdfstring{Console Test App}{Console Test App}}\label{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1666}
If you want to play around with your HID device before starting any development with HIDAPI and using a GUI app is not an option for you, you may try \href{https://github.com/todbot/hidapitester}{\texttt{{\ttfamily hidapitester}}}.

This app has a console interface for most of the features supported by HIDAPI library.\hypertarget{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1667}{}\doxysubsection{\texorpdfstring{What Does the API Look Like?}{What Does the API Look Like?}}\label{dir_7267abf373c38206ba8a7bac64495bfb_autotoc_md1667}

If you want to build your own application that uses HID devices with HIDAPI, you need to get HIDAPI development package.

Depending on what your development environment is, HIDAPI likely to be provided by your package manager.

For instance on Ubuntu, HIDAPI is available via APT:\+
