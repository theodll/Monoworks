SDL can be included in your project in 2 major ways:\+
\begin{DoxyItemize}
\item using a system SDL library, provided by your (UNIX) distribution or a package manager
\item using a vendored SDL library:\+ this is SDL copied or symlinked in a subfolder.
\end{DoxyItemize}

The following CMake script supports both, depending on the value of {\ttfamily MYGAME\+\_\+\+VENDORED}.


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\_minimum\_required(VERSION\ 3.5)}
\DoxyCodeLine{project(mygame)}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ Create\ an\ option\ to\ switch\ between\ a\ system\ sdl\ library\ and\ a\ vendored\ SDL\ library}
\DoxyCodeLine{option(MYGAME\_VENDORED\ "{}Use\ vendored\ libraries"{}\ OFF)}
\DoxyCodeLine{}
\DoxyCodeLine{if(MYGAME\_VENDORED)}
\DoxyCodeLine{\ \ \ \ \#\ This\ assumes\ you\ have\ added\ SDL\ as\ a\ submodule\ in\ vendored/SDL}
\DoxyCodeLine{\ \ \ \ add\_subdirectory(vendored/SDL\ EXCLUDE\_FROM\_ALL)}
\DoxyCodeLine{else()}
\DoxyCodeLine{\ \ \ \ \#\ 1.\ Look\ for\ a\ SDL3\ package,}
\DoxyCodeLine{\ \ \ \ \#\ 2.\ look\ for\ the\ SDL3-\/shared\ component,\ and}
\DoxyCodeLine{\ \ \ \ \#\ 3.\ fail\ if\ the\ shared\ component\ cannot\ be\ found.}
\DoxyCodeLine{\ \ \ \ find\_package(SDL3\ REQUIRED\ CONFIG\ REQUIRED\ COMPONENTS\ SDL3-\/shared)}
\DoxyCodeLine{endif()}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ Create\ your\ game\ executable\ target\ as\ usual}
\DoxyCodeLine{add\_executable(mygame\ WIN32\ mygame.c)}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ Link\ to\ the\ actual\ SDL3\ library.}
\DoxyCodeLine{target\_link\_libraries(mygame\ PRIVATE\ SDL3::SDL3)}

\end{DoxyCode}

\end{DoxyCode}

\begin{DoxyCode}{0}

\end{DoxyCode}
\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-cmake_autotoc_md1220}{}\doxysubsection{\texorpdfstring{Shared or static}{Shared or static}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-cmake_autotoc_md1220}
By default, only a dynamic (=shared) SDL library is built and installed. The options {\ttfamily -\/DSDL\+\_\+\+SHARED=} and {\ttfamily -\/DSDL\+\_\+\+STATIC=} accept boolean values to change this.

Exceptions exist:\+
\begin{DoxyItemize}
\item some platforms don\textquotesingle{}t support dynamic libraries, so only {\ttfamily -\/DSDL\+\_\+\+STATIC=ON} makes sense.
\item a static Apple framework is not supported
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-cmake_autotoc_md1221}{}\doxysubsection{\texorpdfstring{Man pages}{Man pages}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-cmake_autotoc_md1221}

 On Unix systems, {\ttfamily \$\texorpdfstring{$<$}{<}TARGET\+\_\+\+FILE:\+...\texorpdfstring{$>$}{>}} will refer to the dynamic library (or framework), and you might need to use {\ttfamily \$\texorpdfstring{$<$}{<}TARGET\+\_\+\+SONAME\+\_\+\+FILE:\+tgt\texorpdfstring{$>$}{>}} instead.

Most often, you can avoid copying libraries by configuring your project with absolute \href{https://cmake.org/cmake/help/latest/variable/CMAKE_LIBRARY_OUTPUT_DIRECTORY.html}{\texttt{{\ttfamily CMAKE\+\_\+\+LIBRARY\+\_\+\+OUTPUT\+\_\+\+DIRECTORY}}} and \href{https://cmake.org/cmake/help/latest/variable/CMAKE_RUNTIME_OUTPUT_DIRECTORY.html}{\texttt{{\ttfamily CMAKE\+\_\+\+RUNTIME\+\_\+\+OUTPUT\+\_\+\+DIRECTORY}}} paths. When using a multi-\/config generator (such as Visual Studio or Ninja Multi-\/\+Config), eventually add {\ttfamily /\+\$\texorpdfstring{$<$}{<}CONFIG\texorpdfstring{$>$}{>}} to both paths.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-cmake_autotoc_md1237}{}\doxysubsection{\texorpdfstring{Linking against a static SDL library fails due to relocation errors}{Linking against a static SDL library fails due to relocation errors}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-cmake_autotoc_md1237}
On unix platforms, all code that ends up in shared libraries needs to be built as relocatable (=position independent) code. However, by default CMake builds static libraries as non-\/relocatable. Configuring SDL with {\ttfamily -\/DCMAKE\+\_\+\+POSITION\+\_\+\+INDEPENDENT\+\_\+\+CODE=ON} will result in a static {\ttfamily lib\+SDL3.\+a} library which you can link against to create a shared library.
