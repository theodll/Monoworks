Higher resolutions (640×480 and above) require a VESA VBE 1.\+2+ compatible video card.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1290}{}\doxysection{\texorpdfstring{Notes}{Notes}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1290}
\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1291}{}\doxysubsection{\texorpdfstring{Memory Model}{Memory Model}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1291}
The DOS port produces 32-\/bit protected-\/mode DPMI executables. It uses the "{}fat DS"{} nearptr trick ({\ttfamily \+\_\+\+\_\+djgpp\+\_\+nearptr\+\_\+enable()}) to convert physical addresses to usable C pointers. This allows direct access to the VESA linear framebuffer and DMA buffers from C code without segment descriptor manipulation.

SDL on DOS requires the fat DS trick. {\ttfamily SDL\+\_\+\+Run\+App()} enables it automatically via {\ttfamily \doxylink{_s_d_l__main_8h_source}{SDL\+\_\+main.\+h}}. If you define {\ttfamily SDL\+\_\+\+MAIN\+\_\+\+HANDLED}, you must call {\ttfamily \+\_\+\+\_\+djgpp\+\_\+nearptr\+\_\+enable()} yourself before initializing SDL, or video initialization will fail with a clear error message.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1292}{}\doxysubsection{\texorpdfstring{Threading}{Threading}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1292}
DOS has no OS-\/level threads. SDL on DOS implements cooperative threading via a mini-\/scheduler that uses {\ttfamily setjmp}/\+{\ttfamily longjmp} for context switching. Threads are never preempted mid-\/instruction. Context switches occur only at explicit yield points such as {\ttfamily SDL\+\_\+\+Delay} and the event pump, so make sure your main loop calls {\ttfamily SDL\+\_\+\+Pump\+Events} or {\ttfamily SDL\+\_\+\+Delay} regularly. {\ttfamily SDL\+\_\+\+Delay(0)} yields to other threads without sleeping and is safe to call in tight loops. In some cases, like a load screen, a longer delay (e.\+g. {\ttfamily SDL\+\_\+\+Delay(16)}) may be needed to give background threads enough CPU time.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1293}{}\doxysubsection{\texorpdfstring{Video}{Video}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1293}
The video driver supports VGA mode 13h (320×200×256) on any VGA card, and higher resolutions via VESA BIOS Extensions (VBE 1.\+2+). Both linear framebuffer (VBE 2.\+0+) and banked framebuffer modes are supported. Hardware page-\/flipping is used for tear-\/free rendering when available.

Only software rendering is supported. There is no GPU renderer.

All video modes are effectively fullscreen. When creating a window, the driver selects the closest available video mode to the requested size.

8-\/bit indexed color (INDEX8) modes with programmable VGA DAC palettes are supported but will only be used when explicitly requested via the {\ttfamily SDL\+\_\+\+PIXELFORMAT} window creation property.

EGA and CGA cards are not supported. The driver detects VGA hardware at initialization and will fail with a clear error message if VGA is not present.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1294}{}\doxysubsubsection{\texorpdfstring{Direct Framebuffer Hint}{Direct Framebuffer Hint}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1294}
Setting {\ttfamily SDL\+\_\+\+HINT\+\_\+\+DOS\+\_\+\+ALLOW\+\_\+\+DIRECT\+\_\+\+FRAMEBUFFER} to {\ttfamily "{}1"{}} before calling {\ttfamily SDL\+\_\+\+Get\+Window\+Surface()} enables a fast path that skips the normal surface copy. {\ttfamily SDL\+\_\+\+Update\+Window\+Surface()} copies the system-\/\+RAM surface directly to VRAM via {\ttfamily dosmemput} and only programs the VGA DAC palette when it changes. Page flipping is done without waiting for vblank. No software cursor compositing is performed.

This mode is designed for applications like Quake that manage their own rendering and want maximum frame throughput. The trade-\/offs are:\+


\begin{DoxyItemize}
\item No vsync. Tearing is expected.
\item No software cursor. The application must draw its own cursor if needed.
\item Reading back the surface may be slow on real hardware (uncached VRAM).
\end{DoxyItemize}

The hint must be set before the first call to {\ttfamily SDL\+\_\+\+Get\+Window\+Surface()}. Changing it after that has no effect.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1295}{}\doxysubsection{\texorpdfstring{Audio}{Audio}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1295}
\doxylink{struct_sound}{Sound} Blaster support is available for SB16 (16-\/bit stereo), SB Pro (8-\/bit stereo), and SB 2.\+0/\+1.x (8-\/bit mono). Configured automatically from the {\ttfamily BLASTER} environment variable.

A ring buffer sits between the SDL audio pipeline and the DMA hardware. The audio thread fills the ring buffer cooperatively, and the \doxylink{struct_sound}{Sound} Blaster IRQ handler copies data from the ring buffer to the DMA buffer directly. This gives roughly 45 ms of cushion before audio would stutter. If your game runs at 22 fps or above, audio will be glitch-\/free with no extra effort. Below 20 fps, adding a {\ttfamily SDL\+\_\+\+Delay(0)} in the middle of your game loop should be enough to keep the ring buffer fed.

Audio recording is not implemented.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1296}{}\doxysubsection{\texorpdfstring{Input}{Input}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1296}

\begin{DoxyItemize}
\item Keyboard:\+ IRQ1-\/driven with full extended scancode (0x\+E0 prefix) support.
\item Mouse:\+ INT 33h mouse driver with relative motion via mickeys.
\item Joystick:\+ gameport joystick via BIOS INT 15h (axes) and direct port 0x201 reads (buttons) with software calibration.
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1297}{}\doxysubsection{\texorpdfstring{Limitations}{Limitations}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-dos_autotoc_md1297}
