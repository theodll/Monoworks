Dear Im\+Gui outputs 16-\/bit vertex indices by default. Allegro doesn\textquotesingle{}t support them natively, so we have two solutions:\+ convert the indices manually in imgui\+\_\+impl\+\_\+allegro5.\+cpp, or compile dear imgui with 32-\/bit indices. You can either modify \doxylink{imconfig_8h_source}{imconfig.\+h} that comes with Dear Im\+Gui (easier), or set a C++ preprocessor option IMGUI\+\_\+\+USER\+\_\+\+CONFIG to find to a filename. We are providing {\ttfamily \doxylink{imconfig__allegro5_8h_source}{imconfig\+\_\+allegro5.\+h}} that enables 32-\/bit indices. Note that the backend supports {\itshape BOTH} 16-\/bit and 32-\/bit indices, but 32-\/bit indices will be slightly faster as they won\textquotesingle{}t require a manual conversion.\hypertarget{dir_c69e5721b279e0a82b22a029741aab1b_autotoc_md710}{}\doxysubsection{\texorpdfstring{How to Build}{How to Build}}\label{dir_c69e5721b279e0a82b22a029741aab1b_autotoc_md710}
