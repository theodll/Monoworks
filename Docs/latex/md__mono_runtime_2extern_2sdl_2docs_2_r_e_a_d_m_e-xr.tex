This document covers how to build Open\+XR (VR/\+\+AR) applications using SDL\textquotesingle{}s GPU API with Open\+XR integration.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-xr_autotoc_md1487}{}\doxysection{\texorpdfstring{Overview}{Overview}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-xr_autotoc_md1487}
SDL3 provides Open\+XR integration through the GPU API, allowing you to render to VR/\+\+AR headsets using a unified interface across multiple graphics backends (Vulkan, D3\+D12, Metal).

{\bfseries{Key features:\+}}
\begin{DoxyItemize}
\item Automatic Open\+XR instance and session management
\item Swapchain creation and image acquisition
\item Support for multi-\/pass stereo rendering
\item Works with desktop VR runtimes (Steam\+VR, Oculus, Windows Mixed Reality) and standalone headsets (Meta Quest, Pico)
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-xr_autotoc_md1488}{}\doxysection{\texorpdfstring{Desktop Development}{Desktop Development}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-xr_autotoc_md1488}

\begin{DoxyEnumerate}
\item {\bfseries{Open\+XR Loader}} ({\ttfamily openxr\+\_\+loader.\+dll} /\+ {\ttfamily libopenxr\+\_\+loader.\+so})
\begin{DoxyItemize}
\item On Windows:\+ Usually installed with VR runtime software (Oculus, Steam\+VR)
\item On Linux:\+ Install via package manager (e.\+g., {\ttfamily libopenxr-\/loader1} on Ubuntu)
\item Can also use {\ttfamily SDL\+\_\+\+HINT\+\_\+\+OPENXR\+\_\+\+LIBRARY} to specify a custom loader path
\end{DoxyItemize}
\item {\bfseries{Open\+XR Runtime}}
\begin{DoxyItemize}
\item At least one Open\+XR runtime must be installed and active
\item Examples:\+ Steam\+VR, Oculus Desktop, Monado (Linux)
\end{DoxyItemize}
\item {\bfseries{VR Headset}}
\begin{DoxyItemize}
\item Connected and recognized by the runtime
\end{DoxyItemize}
