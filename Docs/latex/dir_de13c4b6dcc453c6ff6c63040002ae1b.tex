Demonstrates that unpatched Tracy crashes when a profiler connects to a HIP application built with {\ttfamily TRACY\+\_\+\+ON\+\_\+\+DEMAND} and {\ttfamily TRACY\+\_\+\+ROCPROF}.\hypertarget{dir_de13c4b6dcc453c6ff6c63040002ae1b_autotoc_md1149}{}\doxysubsection{\texorpdfstring{Root cause}{Root cause}}\label{dir_de13c4b6dcc453c6ff6c63040002ae1b_autotoc_md1149}
Three bugs in {\ttfamily Tracy\+Rocprof.\+cpp} break on-\/demand profiling:\+


\begin{DoxyEnumerate}
\item {\bfseries{Gpu\+New\+Context not deferred.}} {\ttfamily gpu\+\_\+context\+\_\+allocate()} writes a {\ttfamily Gpu\+New\+Context} queue item but does not call {\ttfamily Defer\+Item()}. When a Tracy client connects late, the context creation message is never replayed. The server then receives {\ttfamily Gpu\+Zone\+Begin} events for a context it has never seen, triggering:\+ \begin{DoxyVerb}Assertion `ctx' failed in ProcessGpuZoneBeginImplCommon
\end{DoxyVerb}

\end{DoxyEnumerate}


\begin{DoxyEnumerate}
\item {\bfseries{Gpu\+Context\+Name not deferred.}} Same function writes the context name ("{}rocprofv3"{}) without calling {\ttfamily Defer\+Item()}. Even after fixing bug 1, a late-\/connecting client sees the GPU context but it appears unnamed in the profiler. Use {\ttfamily check\+\_\+gpu\+\_\+ctx\+\_\+name} to verify.
\item {\bfseries{Kernel symbols dropped before init.}} The {\ttfamily data-\/\texorpdfstring{$>$}{>}\doxylink{group__init}{Initialization, version and error reference}} guard at the top of {\ttfamily tool\+\_\+callback\+\_\+tracing\+\_\+callback()} blocks all callbacks before the GPU context is allocated. Kernel symbol registrations ({\ttfamily CODE\+\_\+\+OBJECT\+\_\+\+DEVICE\+\_\+\+KERNEL\+\_\+\+SYMBOL\+\_\+\+REGISTER}) happen at HIP init time — before {\ttfamily data-\/\texorpdfstring{$>$}{>}\doxylink{group__init}{Initialization, version and error reference}} is true — so they are silently dropped. Even if the crash is worked around, kernel names would be missing.
\end{DoxyEnumerate}\hypertarget{dir_de13c4b6dcc453c6ff6c63040002ae1b_autotoc_md1150}{}\doxysubsection{\texorpdfstring{Prerequisites}{Prerequisites}}\label{dir_de13c4b6dcc453c6ff6c63040002ae1b_autotoc_md1150}
