Some aspects of SDL functionality are common to all Unix-\/based platforms.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-platforms_autotoc_md1427}{}\doxysubsection{\texorpdfstring{\label{_setuid}%
Privileged processes (setuid, setgid, setcap)}{\label{_setuid}%
Privileged processes (setuid, setgid, setcap)}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-platforms_autotoc_md1427}
SDL is not designed to be used in programs with elevated privileges, such as setuid ({\ttfamily chmod u+s}) or setgid ({\ttfamily chmod g+s}) executables, or executables with file-\/based capabilities ({\ttfamily setcap cap\+\_\+sys\+\_\+nice+ep} or similar). It does not make any attempt to avoid trusting environment variables or other aspects of the inherited execution environment. Programs running with elevated privileges in an attacker-\/controlled execution environment should not call SDL functions.
