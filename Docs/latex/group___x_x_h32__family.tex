\doxysubsubsection{Detailed Description}
Contains functions used in the classic 32-\/bit xx\+Hash algorithm.

\begin{DoxyNote}{Note}
XXH32 is useful for older platforms, with no or poor 64-\/bit performance. Note that the \doxylink{group___x_x_h3__family}{XXH3 family} provides competitive speed for both 32-\/bit and 64-\/bit systems, and offers true 64/\+128 bit hash results.
\end{DoxyNote}

typedef struct XXH32\+\_\+state\+\_\+s \doxymbox{\hyperlink{group___x_x_h32__family_ga0048f07d4a3051c20598464f007aa6c5}{XXH32\+\_\+state\+\_\+t}}}



The opaque state struct for the XXH32 streaming API. 

Streaming functions generate the xx\+Hash value from an incremental input. This method is slower than single-\/call functions, due to state management. For small inputs, prefer {\ttfamily \doxylink{group___x_x_h32__family_ga49360c30d0eef2cd4e20cabfe6ab1c55}{XXH32()}} and {\ttfamily \doxylink{group___x_x_h64__family_ga582b359c2b7a04d83837f58e72e60a09}{XXH64()}}, which are better optimized.

An XXH state must first be allocated using {\ttfamily XXH\texorpdfstring{$\ast$}{*}\+\_\+create\+State()}.

Start a new hash by initializing the state with a seed using {\ttfamily XXH\texorpdfstring{$\ast$}{*}\+\_\+reset()}.

Then, feed the hash state by calling {\ttfamily XXH\texorpdfstring{$\ast$}{*}\+\_\+update()} as many times as necessary.

The function returns an error code, with 0 meaning OK, and any other value meaning there is an error.

Finally, a hash value can be produced anytime, by using {\ttfamily XXH\texorpdfstring{$\ast$}{*}\+\_\+digest()}. This function returns the nn-\/bits hash as an int or long long.

It\textquotesingle{}s still possible to continue inserting input into the hash state after a digest, and generate new hash values later on by invoking {\ttfamily XXH\texorpdfstring{$\ast$}{*}\+\_\+digest()}.

When done, release the state using {\ttfamily XXH\texorpdfstring{$\ast$}{*}\+\_\+free\+State()}.
