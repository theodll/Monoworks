\doxysubsection{Detailed Description}
\hypertarget{include_2_s_d_l3_2_s_d_l__dialog_8h_autotoc_md1556}{}\doxysubsection{\texorpdfstring{Category\+Dialog}{Category\+Dialog}}\label{include_2_s_d_l3_2_s_d_l__dialog_8h_autotoc_md1556}
File dialog support.

SDL offers file dialogs, to let users select files with native GUI interfaces. There are "{}open"{} dialogs, "{}save"{} dialogs, and folder selection dialogs. The app can control some details, such as filtering to specific files, or whether multiple files can be selected by the user.

Note that launching a file dialog is a non-\/blocking operation; control returns to the app immediately, and a callback is called later (possibly in another thread) when the user makes a choice. An entry for filters for file dialogs.

{\ttfamily name} is a user-\/readable label for the filter (for example, "{}\+Office document"{}).

{\ttfamily pattern} is a semicolon-\/separated list of file extensions (for example, "{}doc;docx"{}). File extensions may only contain alphanumeric characters, hyphens, underscores and periods. Alternatively, the whole string can be a single asterisk ("{}\texorpdfstring{$\ast$}{*}"{}), which serves as an "{}\+All files"{} filter.
