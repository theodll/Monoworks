Without getting deep into the history, Min\+GW is a long running project that aims to bring gcc to Windows. That said, there\textquotesingle{}s many distributions, versions, and forks floating around. We recommend installing \href{https://www.msys2.org/}{\texttt{MSYS2}}, as it\textquotesingle{}s the easiest way to get a modern toolchain with a package manager to help with dependency management. This would allow you to follow the MSYS2 section below.

Otherwise you\textquotesingle{}ll want to follow the "{}\+Other Distributions"{} section below.

We\textquotesingle{}ll start by creating a simple project to build and run \href{hello.c}{\texttt{hello.\+c}}.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1171}{}\doxysection{\texorpdfstring{MSYS2}{MSYS2}}\label{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1171}

Things can get quite complicated with other distributions of Min\+GW. If you can\textquotesingle{}t follow \doxysectlink{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-cmake}{the cmake intro}{0}, perhaps due to issues getting cmake to understand your toolchain, this section should work.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1176}{}\doxysubsection{\texorpdfstring{Acquire SDL}{Acquire SDL}}\label{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1176}
Download the {\ttfamily SDL3-\/devel-\/\texorpdfstring{$<$}{<}version\texorpdfstring{$>$}{>}-\/mingw.\+zip} asset from \href{https://github.com/libsdl-org/SDL/releases/latest}{\texttt{the latest release.}} Then extract it inside your project folder such that the output of {\ttfamily ls SDL3-\/\texorpdfstring{$<$}{<}version\texorpdfstring{$>$}{>}} looks like {\ttfamily INSTALL.\+md LICENSE.\+txt Makefile README.\+md cmake i686-\/w64-\/mingw32 x86\+\_\+64-\/w64-\/mingw32}.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1177}{}\doxysubsection{\texorpdfstring{Know your Target Architecture}{Know your Target Architecture}}\label{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1177}
It is not uncommon for folks to not realize their distribution is targeting 32bit Windows despite things like the name of the toolchain, or the fact that they\textquotesingle{}re running on a 64bit system. We\textquotesingle{}ll ensure we know up front what we need:\+

Create a file named {\ttfamily arch.\+c} with the following contents:\+ 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\#include\ <stddef.h>}}
\DoxyCodeLine{\textcolor{preprocessor}{\#include\ <stdio.h>}}
\DoxyCodeLine{\textcolor{keywordtype}{int}\ main()\ \{}
\DoxyCodeLine{\textcolor{preprocessor}{\ \ \ \ \#if\ defined(\_\_x86\_64\_\_)\ ||\ defined(\_M\_X64)\ ||\ defined(i386)\ ||\ defined(\_\_i386\_\_)\ ||\ defined(\_\_i386)\ ||\ defined(\_M\_IX86)}}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \textcolor{keywordtype}{size\_t}\ ptr\_size\ =\ \textcolor{keyword}{sizeof}(\textcolor{keywordtype}{int}*);}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \textcolor{keywordflow}{if}\ (4\ ==\ ptr\_size)\ puts(\textcolor{stringliteral}{"{}i686-\/w64-\/mingw32"{}});}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \textcolor{keywordflow}{else}\ \textcolor{keywordflow}{if}\ (8\ ==\ ptr\_size)\ puts(\textcolor{stringliteral}{"{}x86\_64-\/w64-\/mingw32"{}});}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \textcolor{keywordflow}{else}\ puts(\textcolor{stringliteral}{"{}Unknown\ Architecture"{}});}
\DoxyCodeLine{\textcolor{preprocessor}{\ \ \ \ \#else}}

\DoxyCodeLine{\textcolor{preprocessor}{\ \ \ \ \#endif}}
\DoxyCodeLine{\ \ \ \ \textcolor{keywordflow}{return}\ 0;}
\DoxyCodeLine{\}}

\end{DoxyCode}


Then run


\begin{DoxyCode}{0}
\DoxyCodeLine{gcc\ arch.c}
\DoxyCodeLine{./a.exe}

\end{DoxyCode}


This should print out which library directory we\textquotesingle{}ll need to use when compiling, keep this value in mind, you\textquotesingle{}ll need to use it when compiling in the next section as {\ttfamily \texorpdfstring{$<$}{<}arch\texorpdfstring{$>$}{>}}. If you get "{}\+Unknown Architecture"{} please \href{https://github.com/libsdl-org/SDL/issues}{\texttt{report a bug}}.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1178}{}\doxysubsection{\texorpdfstring{Build and Run}{Build and Run}}\label{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw_autotoc_md1178}
Now we should have everything needed to compile and run our program. You\textquotesingle{}ll need to ensure to replace {\ttfamily \texorpdfstring{$<$}{<}version\texorpdfstring{$>$}{>}} with the version of the release of SDL3 you downloaded, as well as use the {\ttfamily \texorpdfstring{$<$}{<}arch\texorpdfstring{$>$}{>}} we learned in the previous section.
