{\itshape (You may browse this at \href{https://github.com/ocornut/imgui/blob/master/docs/FONTS.md}{\texttt{https:\+/\+/\+github.\+com/\+ocornut/\+imgui/\+blob/\+master/\+docs/\+\+FONTS.\+md}} or view this file with any Markdown viewer)}\hypertarget{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md631}{}\doxysubsection{\texorpdfstring{Dear Im\+Gui:\+ Using Fonts}{Dear Im\+Gui:\+ Using Fonts}}\label{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md631}
The code in Dear Im\+Gui embeds a copy of \href{https://github.com/bluescan/proggyfonts}{\texttt{Proggy\+Clean.\+ttf}} by Tristan Grimmer, a 13 pixels high, pixel-\/perfect font used by default. Proggy\+Clean does not scale very nicely.

The code in Dear Im\+Gui embeds a partial copy of \href{https://github.com/ocornut/proggyforever}{\texttt{Proggy\+Forever.\+ttf}} by Disco Hello \& Tristan Grimmer, a new font mimicking Proggy\+Clean which does scale nicely.

We embed fonts in the code so you can use Dear Im\+Gui without any file system access. If you don\textquotesingle{}t use them you can set {\ttfamily IMGUI\+\_\+\+DISABLE\+\_\+\+DEFAULT\+\_\+\+FONT} in your \href{https://github.com/ocornut/imgui/blob/master/imconfig.h}{\texttt{imconfig.\+h}} file to ship binaries without the fonts and save about \texorpdfstring{$\sim$}{\string~}26 KB.

Calling io.\+Fonts-\/\texorpdfstring{$>$}{>}Add\+Font\+Default\+Vector() loads Proggy\+Forever. Calling io.\+Fonts-\/\texorpdfstring{$>$}{>}Add\+Font\+Default\+Bitmap() loads Proggy\+Clean. Calling io.\+Fonts-\/\texorpdfstring{$>$}{>}Add\+Font\+Default() selects one based on the expected default font size (when {\ttfamily style.\+Font\+Size\+Base \texorpdfstring{$\ast$}{*} style.\+Font\+Scale\+Main \texorpdfstring{$\ast$}{*} style.\+Font\+Size\+Dpi \texorpdfstring{$>$}{>}= 15} we use Proggy\+Forever).

You may also load external .TTF/\+.OTF files, see instructions on this page. In the \href{https://github.com/ocornut/imgui/tree/master/misc/fonts}{\texttt{misc/\+fonts/\+}} folder you can find a few suggested fonts, provided as a convenience.

Use the encoding viewer to confirm encoding of string literal in your source code is correct.\hypertarget{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md637}{}\doxysubsubsection{\texorpdfstring{(3) Missing glyph ranges.}{(3) Missing glyph ranges.}}\label{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md637}
🆕 {\bfseries{Since 1.\+92, with an up to date backend:\+ specifying glyph ranges is unnecessary.}}

⏪ Before 1.\+92:\+ you need to load a font with explicit glyph ranges if you want to use non-\/\+ASCII characters. See Fonts Loading InstructionsFonts Loading Instructions. Use Debug ToolsDebug Tools confirm loaded fonts and loaded glyph ranges.

This was a previous constraint of Dear Im\+Gui (lifted in 1.\+92):\+ when loading a font you need to specify which characters glyphs to load. All loaded fonts glyphs are rendered into a single texture atlas ahead of time. Calling either of {\ttfamily io.\+Fonts-\/\texorpdfstring{$>$}{>}Get\+Tex\+Data\+As\+Alpha8()}, {\ttfamily io.\+Fonts-\/\texorpdfstring{$>$}{>}Get\+Tex\+Data\+As\+RGBA32()} or {\ttfamily io.\+Fonts-\/\texorpdfstring{$>$}{>}Build()} will build the atlas. This is generally called by the Renderer backend, e.\+g. {\ttfamily Im\+Gui\+\_\+\+Impl\+DX11\+\_\+\+New\+Frame()} calls it. {\bfseries{If you use custom glyphs ranges, make sure the array is persistent}} and available during the calls to {\ttfamily Get\+Tex\+Data\+As\+Alpha8()/\+\+Get\+Tex\+Data\+As\+RGBA32()/\+\+Build()}.\hypertarget{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md638}{}\doxysubsubsection{\texorpdfstring{(4) Font atlas texture fails to upload to GPU.}{(4) Font atlas texture fails to upload to GPU.}}\label{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md638}
🆕 {\bfseries{Since 1.\+92, with an up to date backend:\+ atlas is built incrementally and dynamically resized, this is less likely to happen}}

:\+rewind:\+ This is often of byproduct of point 3. If you have large number of glyphs or multiple fonts, the texture may become too big for your graphics API. {\bfseries{The typical result of failing to upload a texture is if every glyph or everything appears as empty white rectangles.}} Mind the fact that some graphics drivers have texture size limitation. If you are building a PC application, mind the fact that your users may use hardware with lower limitations than yours.



Some solutions:\+
\begin{DoxyItemize}
\item You may reduce oversampling, e.\+g. {\ttfamily font\+\_\+config.\+OversampleH = 1}, this will half your texture size for a quality loss. Note that while OversampleH = 2 looks visibly very close to 3 in most situations, with OversampleH = 1 the quality drop will be noticeable. Read about oversampling \href{https://github.com/nothings/stb/blob/master/tests/oversample}{\texttt{here}}.
\item Reduce glyphs ranges by calculating them from source localization data. You can use the {\ttfamily \doxylink{struct_im_font_glyph_ranges_builder}{Im\+Font\+Glyph\+Ranges\+Builder}} for this purpose and rebuilding your atlas between frames when new characters are needed. This will be the biggest win!
\item Set {\ttfamily io.\+Fonts.\+Flags \texorpdfstring{$\vert$}{|}= Im\+Font\+Atlas\+Flags\+\_\+\+No\+Power\+Of\+Two\+Height;} to disable rounding the texture height to the next power of two.
\end{DoxyItemize}\hypertarget{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md639}{}\doxysubsubsection{\texorpdfstring{(5) Reduce texture resizes/\+copy on startup}{(5) Reduce texture resizes/\+copy on startup}}\label{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md639}
🆕 Since 1.\+92, the \doxylink{struct_im_font_atlas}{Im\+Font\+Atlas} is initially created using a 512x128 texture, then grows as glyphs and fonts are used. Atlas growth leads to alloc+copy, and both the old and new sized textures are present in memory for a short time (typically one frame). If you use known fonts and want to reduce initial growth, you may set {\ttfamily Tex\+Min\+Width} and {\ttfamily Tex\+Min\+Height} during initializaton.

\hypertarget{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md642}{}\doxysubsection{\texorpdfstring{New! Dynamic Fonts system in 1.\+92 (June 2025)}{New! Dynamic Fonts system in 1.\+92 (June 2025)}}\label{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md642}
v1.\+92 introduces a newer, dynamic font system. It requires backend to support the {\ttfamily Im\+Gui\+Backend\+Flags\+\_\+\+Has\+Textures} feature:\+
\begin{DoxyItemize}
\item Users of icons, Asian and non-\/\+English languages do not need to pre-\/build all glyphs ahead of time. Saving on loading time, memory, and also reducing issues with missing glyphs. Specifying glyph ranges is not needed anymore.
\item {\ttfamily Push\+Font(\+NULL, new\+\_\+size)} may be used anytime to change font size.
\item Packing custom rectangles is more convenient as pixels may be written to immediately.
\item Any update to fonts previously required backend specific calls to re-\/upload the texture, and said calls were not portable across backends. It is now possible to scale fonts etc. in a way that doesn\textquotesingle{}t require you to make backend-\/specific calls.
\item It is possible to plug a custom loader/\+backend to any font source.
\end{DoxyItemize}

See \href{https://github.com/ocornut/imgui/issues/8465}{\texttt{\#8465}} for more details.\hypertarget{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md643}{}\doxysubsubsubsubsection{\texorpdfstring{\protect\doxysectlink{index}{Return to Index}{0}Return to Index}{{Return to Index}Return to Index}}\label{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md643}

(1) In C/\+\+C++ and most programming languages if you want to use a backslash {\ttfamily \textbackslash{}} within a string literal, you need to write it double backslash {\ttfamily \textbackslash{}\textbackslash{}}. At it happens, Windows uses backslashes as a path separator, so be mindful. 
\begin{DoxyCode}{0}
\DoxyCodeLine{io.Fonts-\/>AddFontFromFileTTF(\textcolor{stringliteral}{"{}MyFiles\(\backslash\)MyImage01.jpg"{}},\ ...);\ \ \ \textcolor{comment}{//\ This\ is\ INCORRECT!!}}
\DoxyCodeLine{io.Fonts-\/>AddFontFromFileTTF(\textcolor{stringliteral}{"{}MyFiles\(\backslash\)\(\backslash\)MyImage01.jpg"{}},\ ...);\ \ \textcolor{comment}{//\ This\ is\ CORRECT}}

\end{DoxyCode}
 In some situations, you may also use {\ttfamily /\+} path separator under Windows.

(2) Make sure your IDE/\+debugger settings starts your executable from the right working (current) directory. In Visual Studio you can change your working directory in project {\ttfamily Properties \texorpdfstring{$>$}{>} General \texorpdfstring{$>$}{>} Debugging \texorpdfstring{$>$}{>} Working Directory}. People assume that their execution will start from the root folder of the project, where by default it often starts from the folder where object or executable files are stored. 
\begin{DoxyCode}{0}
\DoxyCodeLine{io.Fonts-\/>AddFontFromFileTTF(\textcolor{stringliteral}{"{}MyImage01.jpg"{}},\ ...);\ \ \ \ \ \ \ \textcolor{comment}{//\ Relative\ filename\ depends\ on\ your\ Working\ Directory\ when\ running\ your\ program!}}
\DoxyCodeLine{io.Fonts-\/>AddFontFromFileTTF(\textcolor{stringliteral}{"{}../MyImage01.jpg"{}},\ ...);\ \ \ \ \textcolor{comment}{//\ Load\ from\ the\ parent\ folder\ of\ your\ Working\ Directory}}

\end{DoxyCode}

{\bfseries{For non-\/\+ASCII characters display, a common user issue is not passing correctly UTF-\/8 encoded strings.}}

(1) We provide a function {\ttfamily Im\+Gui::\+\+Debug\+Text\+Encoding(const char\texorpdfstring{$\ast$}{*} text)} which you can call to verify the content of your UTF-\/8 strings. This is a convenient way to confirm that your encoding is correct.

\DoxyCodeLine{ImGui::SeparatorText(\textcolor{stringliteral}{"{}INCORRECT"{}});}
\DoxyCodeLine{ImGui::DebugTextEncoding(\textcolor{stringliteral}{"{}こんにちは"{}});}

\end{DoxyCode}
 

You can also find this tool under {\ttfamily Metrics/\+\+Debuggers-\/\texorpdfstring{$>$}{>}Tools-\/\texorpdfstring{$>$}{>}UTF-\/8 Encoding viewer} if you want to paste from clipboard, but this won\textquotesingle{}t validate the UTF-\/8 encoding done by your compiler.

(2) To encode in UTF-\/8:\+

There are also compiler-\/specific ways to enforce UTF-\/8 encoding by default:\+


\begin{DoxyItemize}
\item Visual Studio compiler:\+ {\ttfamily /\+utf-\/8} command-\/line flag.
\item Visual Studio compiler:\+ {\ttfamily \#pragma execution\+\_\+character\+\_\+set("{}utf-\/8"{})} inside your code.
\item Since May 2023 we have changed the Visual Studio projects of all our examples to use {\ttfamily /\+utf-\/8} (\href{https://github.com/ocornut/imgui/commit/513af1efc9080857bbd10000d98f98f2a0c96803}{\texttt{see commit}}).
\end{DoxyItemize}

Or, since C++11, you can use the {\ttfamily u8"{}my text"{}} syntax to encode literal strings as UTF-\/8. e.\+g.:\+ 
\begin{DoxyCode}{0}
\DoxyCodeLine{ImGui::Text(u8\textcolor{stringliteral}{"{}hello"{}});}
\DoxyCodeLine{ImGui::Text(u8\textcolor{stringliteral}{"{}こんにちは"{}});\ \ \ \textcolor{comment}{//\ this\ will\ always\ be\ encoded\ as\ UTF-\/8}}
\DoxyCodeLine{ImGui::Text(\textcolor{stringliteral}{"{}こんにちは"{}});\ \ \ \ \ \textcolor{comment}{//\ the\ encoding\ of\ this\ is\ depending\ on\ compiler\ settings/flags\ and\ may\ be\ incorrect.}}

\end{DoxyCode}


Since C++20, because the C++ committee hate its users, they decided to change the {\ttfamily u8"{}"{}} syntax to not return {\ttfamily const char\texorpdfstring{$\ast$}{*}} but a new type {\ttfamily const char8\+\_\+t\texorpdfstring{$\ast$}{*}} which doesn\textquotesingle{}t cast to {\ttfamily const char\texorpdfstring{$\ast$}{*}}. Because of type usage of {\ttfamily u8"{}"{}} in C++20 is a little more tedious:\+ 
\begin{DoxyCode}{0}
\DoxyCodeLine{ImGui::Text((\textcolor{keyword}{const}\ \textcolor{keywordtype}{char}*)u8\textcolor{stringliteral}{"{}こんにちは"{}});}

\end{DoxyCode}
 However, you can disable this behavior completely using the compiler option \href{https://learn.microsoft.com/en-us/cpp/build/reference/zc-char8-t?view=msvc-170}{\texttt{{\ttfamily /\+\+Zc:\+char8\+\_\+t-\/}}} for MSVC and \href{https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2019/p1423r3.html}{\texttt{{\ttfamily -\/fno-\/char8\+\_\+t}}} for Clang and GCC. \hypertarget{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md680}{}\doxysubsubsubsubsection{\texorpdfstring{\protect\doxysectlink{index}{Return to Index}{0}Return to Index}{{Return to Index}Return to Index}}\label{md__mono_engine_2thirdparty_2imgui_2docs_2_f_o_n_t_s_autotoc_md680}
