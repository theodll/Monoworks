SDL supports a number of development environments:\+
\begin{DoxyItemize}
\item \doxysectlink{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-cmake}{CMake}{0}
\item \doxysectlink{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-visualstudio}{Visual Studio on Windows}{0}
\item \doxysectlink{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-mingw}{gcc on Windows}{0}
\item \doxysectlink{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-xcode}{Xcode on Apple platforms}{0}
\item \doxysectlink{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-androidstudio}{Android Studio}{0}
\item \doxysectlink{md__mono_runtime_2extern_2sdl_2docs_2_i_n_t_r_o-emscripten}{Emscripten for web}{0}
\end{DoxyItemize}

SDL is also usable in other environments. The basic steps are to use CMake to build the library and then use the headers and library that you built in your project. You can search online to see if anyone has specific steps for your setup.\hypertarget{md__mono_runtime_2extern_2sdl_2_i_n_s_t_a_l_l_autotoc_md1619}{}\doxysection{\texorpdfstring{Documentation}{Documentation}}\label{md__mono_runtime_2extern_2sdl_2_i_n_s_t_a_l_l_autotoc_md1619}
