\doxysubsection{Detailed Description}
\hypertarget{dir_ab861387eefa8d506c1a9377ae435055_autotoc_md1671}{}\doxysubsection{\texorpdfstring{pp\+\_\+data\+\_\+dump.\+exe for Windows}{pp\+\_\+data\+\_\+dump.\+exe for Windows}}\label{dir_ab861387eefa8d506c1a9377ae435055_autotoc_md1671}
pp\+\_\+data\+\_\+dump.\+exe is a small command line tool for Windows, which dumps the content of the \href{https://learn.microsoft.com/en-us/windows-hardware/drivers/hid/preparsed-data}{\texttt{Preparsed Data}} structure, provided by the Windows HID subsystem, into a file.

The generated file is in a text format, which is readable for human, as by the hid\+\_\+report\+\_\+reconstructor\+\_\+test.\+exe unit test executable of the HIDAPI project. This unit test allows it to test the HIDAPIs report descriptor reconstructor -\/ offline, without the hardware device connected.

pp\+\_\+data\+\_\+dump.\+exe has no arguments, just connect you HID device and execute pp\+\_\+data\+\_\+dump.\+exe. It will generate one file with the name 
\begin{DoxyCode}{0}
\DoxyCodeLine{<vendor\_id>\_<product\_id>\_<usage>\_<usage\_table>.pp\_data}

\end{DoxyCode}
 for each top-\/level collection, of each connected HID device.\hypertarget{dir_ab861387eefa8d506c1a9377ae435055_autotoc_md1672}{}\doxysubsection{\texorpdfstring{File content}{File content}}\label{dir_ab861387eefa8d506c1a9377ae435055_autotoc_md1672}
The content of such a .pp\+\_\+data file looks like the struct, which HIDAPI uses internally to represent the Preparsed Data.

{\itshape NOTE:\+ Windows parses HID report descriptors into opaque {\ttfamily \+\_\+\+HIDP\+\_\+\+PREPARSED\+\_\+\+DATA} objects. The internal structure of {\ttfamily \+\_\+\+HIDP\+\_\+\+PREPARSED\+\_\+\+DATA} is reserved for internal system use.\textbackslash{} Microsoft doesn\textquotesingle{}t document this structure. \href{https://chromium.googlesource.com/chromium/src/+/73fdaaf605bb60caf34d5f30bb84a417688aa528/services/device/hid/hid_preparsed_data.cc}{\texttt{hid\+\_\+preparsed\+\_\+data.\+cc}} is taken as a reference for its parsing.}
