If you think you have found a bug and would like to report it, here are the steps you should take:\+


\begin{DoxyItemize}
\item Before opening a new issue, ensure your bug has not already been reported on the \href{https://github.com/libsdl-org/SDL/issues}{\texttt{Git\+Hub Issues page}}.
\item On the issue tracker, click on \href{https://github.com/libsdl-org/SDL/issues/new}{\texttt{New Issue}}.
\item Include details about your environment, such as your Operating System and SDL version.
\item If possible, provide a small example that reproduces your bug.
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1241}{}\doxysubsection{\texorpdfstring{Suggesting enhancements}{Suggesting enhancements}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1241}
If you want to suggest changes for the project, here are the steps you should take:\+


\begin{DoxyItemize}
\item Check if the suggestion has already been made on:\+
\begin{DoxyItemize}
\item the \href{https://github.com/libsdl-org/SDL/issues}{\texttt{issue tracker}};
\item the \href{https://discourse.libsdl.org/}{\texttt{discourse forum}};
\item or if a \href{https://github.com/libsdl-org/SDL/pulls}{\texttt{pull request}} already exists.
\end{DoxyItemize}
\item On the issue tracker, click on \href{https://github.com/libsdl-org/SDL/issues/new}{\texttt{New Issue}}.
\item Describe what change you would like to happen.
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1242}{}\doxysection{\texorpdfstring{Contributing code}{Contributing code}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1242}
This section will cover how the process of forking the project, making a change and opening a pull request.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1243}{}\doxysubsection{\texorpdfstring{Forking the project}{Forking the project}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1243}
The first step consists in making a fork of the project, this is only necessary for the first contribution.

Head over to \href{https://github.com/libsdl-org/SDL}{\texttt{https:\+/\+/\+github.\+com/\+libsdl-\/org/\+\+SDL}} and click on the {\ttfamily Fork} button in the top right corner of your screen, you may leave the fields unchanged and click {\ttfamily Create Fork}.

You will be redirected to your fork of the repository, click the green {\ttfamily Code} button and copy the git clone link.

If you had already forked the repository, you may update it from the web page using the {\ttfamily Fetch upstream} button.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1244}{}\doxysubsection{\texorpdfstring{Following the style guide}{Following the style guide}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1244}
Code formatting is done using a custom {\ttfamily .clang-\/format} file, you can learn more about how to run it \href{https://clang.llvm.org/docs/ClangFormat.html}{\texttt{here}}.

Some legacy code may not be formatted, so please avoid formatting the whole file at once and only format around your changes.

For your commit message to be properly displayed on Git\+Hub, it should contain:\+


\begin{DoxyItemize}
\item A short description of the commit of 50 characters or less on the first line.
\item If necessary, add a blank line followed by a long description, each line should be 72 characters or less.
\end{DoxyItemize}

For example:\+


\begin{DoxyCode}{0}
\DoxyCodeLine{Fix\ crash\ in\ SDL\_FooBar.}
\DoxyCodeLine{}
\DoxyCodeLine{This\ addresses\ the\ issue\ \#123456\ by\ making\ sure\ Foo\ was\ successful}
\DoxyCodeLine{before\ calling\ Bar.}

\end{DoxyCode}
\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1245}{}\doxysubsection{\texorpdfstring{Running the tests}{Running the tests}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1245}
Tests allow you to verify if your changes did not break any behaviour, here are the steps to follow:\+


\begin{DoxyItemize}
\item Before pushing, run the {\ttfamily testautomation} suite on your machine, there should be no more failing tests after your change than before.
\item After pushing to your fork, Continuous Integration (Git\+Hub Actions) will ensure compilation and tests still pass on other systems.
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1246}{}\doxysubsection{\texorpdfstring{Opening a pull request}{Opening a pull request}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1246}

\begin{DoxyItemize}
\item Head over to your fork\textquotesingle{}s Git\+Hub page.
\item Click on the {\ttfamily Contribute} button and {\ttfamily Open Pull Request}.
\item Fill out the pull request template.
\item If any changes are requested, you can add new commits to your fork and they will be automatically added to the pull request.
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1247}{}\doxysubsection{\texorpdfstring{Continuous integration}{Continuous integration}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1247}
For each push and/\+or pull request, Git\+Hub Actions will try to build SDL and the test suite on most supported platforms.

Its behaviour can be influenced slightly by including SDL-\/specific tags in your commit message:\+
\begin{DoxyItemize}
\item {\ttfamily \+[sdl-\/ci-\/filter GLOB]\+} limits the platforms for which to run ci.
\item {\ttfamily \+[sdl-\/ci-\/artifacts]\+} forces SDL artifacts, which can then be downloaded from the summary page.
\item {\ttfamily \+[sdl-\/ci-\/trackmem-\/symbol-\/names]\+} makes sure the final report generated by {\ttfamily -\/-\/trackmem} contains symbol names.
\item {\ttfamily \+[sdl-\/ci-\/ctest-\/args]\+} allows overriding the ctest arguments. Adding {\ttfamily -\/-\/repeat-\/until-\/fail N} and {\ttfamily -\/R NAME} is useful.
\end{DoxyItemize}\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1248}{}\doxysection{\texorpdfstring{Contributing to the documentation}{Contributing to the documentation}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1248}

As such, all modifications to syntax; function parameters; return value; version; related functions should be done in the header directly.\hypertarget{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1250}{}\doxysubsection{\texorpdfstring{Editing the wiki}{Editing the wiki}}\label{md__mono_runtime_2extern_2sdl_2docs_2_r_e_a_d_m_e-contributing_autotoc_md1250}
